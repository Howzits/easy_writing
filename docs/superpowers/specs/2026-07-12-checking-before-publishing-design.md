# 文章发布前检查 Skill 设计

## 目标

创建 `checking-before-publishing` Skill，用于微信公众号文章发布前的最后检查。它只修正明确的文字错误，不承担润色、改写或内容审查；同时基于正文生成候选标题和无文字封面图提示词。

## 使用场景

当用户要求“发布前检查”“终稿校对”“只改错别字和标点”“生成公众号标题”或“生成公众号封面提示词”时触发。输入可以是直接粘贴的正文，也可以是可读取的文章文件。

## 检查边界

允许修改：

- 明确的错别字。
- 明显的漏字或赘字，且原本意图清晰、不修正就无法形成完整表达。
- 中文标点的错用、漏用、重复、全半角混用及成对符号不闭合。

禁止修改：

- 不润色措辞，不替换同义词，不追求“更优美”。
- 不调整句式、段落、语序、语气或作者风格。
- 不改变观点、事实、数据、引文和专有名词。
- 不为了消除疑似语病而猜测作者意图。

无法确定的内容保留原文，列入“待作者确认”。如果正文无须修改，明确说明“未发现明确的错别字或标点错误”。

## 工作流程

1. 完整读取正文，识别文章主题、核心判断、目标读者和情绪基调。
2. 逐句检查错别字、明显漏字或赘字及标点问题。
3. 对每一处拟修改内容复核：若会影响含义、风格或事实，则撤销修改并列入“待作者确认”。
4. 输出校对后的完整正文，并列出可核对的修改清单。
5. 仅依据正文生成 10 个标题，覆盖利益点、反差、问题、洞察等不同角度，但不得虚构信息或制造正文不支持的承诺。
6. 生成 3 组差异明确的公众号封面图提示词。

## 固定输出结构

### 一、校对后的正文

保留原始标题层级、段落、换行和整体结构，只应用允许范围内的修改。

### 二、修改清单

使用“原文 → 修改后 → 原因”的形式逐项列出。没有修改时明确说明。

### 三、待作者确认

只列有疑问且无法安全修改的内容，并简述疑点。没有时写“无”。

### 四、候选标题

提供 10 个贴合正文的微信公众号候选标题。标题应有传播力，但避免标题党、夸张承诺、虚构数字或正文没有的结论。

### 五、公众号封面提示词

提供 3 组可直接用于图像生成的中文提示词。每组包含主体、场景、构图、光线、色彩、氛围、视觉风格和留白安排，并统一明确：

- 横版构图，宽高比 `2.35:1`。
- 不包含文章标题。
- 画面中不出现任何文字、字母、数字、Logo 或水印。
- 主题与正文一致，三组方案的视觉方向应明显不同。

## 文件结构

Skill 放在当前仓库根目录下：

```text
checking-before-publishing/
├── SKILL.md
└── agents/
    └── openai.yaml
```

该工作流主要依赖语言判断，不需要脚本、参考资料或素材目录。

## 验收标准

- Skill 元数据能够覆盖发布前检查、错别字、标点、公众号标题和封面提示词等触发语。
- 校对结果不出现措辞润色、句式重写、段落调整或观点变化。
- 每项正文修改均能在修改清单中找到对应记录。
- 不确定项不擅自修改。
- 标题全部能从正文找到依据。
- 三组封面提示词均不含标题文字，明确无任何文字并指定 `2.35:1`。
- 通过 Skill 结构校验和代表性样例测试。
