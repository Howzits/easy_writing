# 中文 README Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `reviewing-article-quality` SKILL 编写一份面向 Codex 和 Claude Code 用户的中文安装与使用说明。

**Architecture:** 在项目根目录新增单一入口 `README.md`，共享项目介绍、能力说明和使用示例，并将平台差异限制在各自的安装小节。所有功能描述以现有 `SKILL.md` 和 `references/rubric.md` 为准。

**Tech Stack:** Markdown、POSIX shell、Git

## Global Constraints

- Codex 用户级安装目录为 `$HOME/.agents/skills/reviewing-article-quality`。
- Claude Code 用户级安装目录为 `$HOME/.claude/skills/reviewing-article-quality`，项目级目录为 `.claude/skills/reviewing-article-quality`。
- 不修改 `reviewing-article-quality/` 中的审查逻辑。
- 安装命令默认不得覆盖已有同名目录。

- [x] **Step 1: 编写 README**

  创建 `README.md`，依次包含项目简介、主要能力、目录结构、Codex 安装、Claude Code 安装、使用示例、输出结构、更新、卸载和注意事项。安装命令使用 `git clone` 加 `cp -R`，复制前通过 `[ -e "$TARGET" ]` 检测同名目标，存在时退出并提示用户先备份或执行更新步骤。

- [x] **Step 2: 检查内容与源文件一致**

  运行：

  ```bash
  rg -n 'Codex|Claude Code|安装|使用|更新|卸载|SKILL.md|rubric.md|openai.yaml' README.md
  ```

  预期：每个主题均至少命中一次，所有文件名与仓库实际结构一致。

- [x] **Step 3: 模拟两个平台的复制安装**

  运行：

  ```bash
  TMP_ROOT="$(mktemp -d)"
  mkdir -p "$TMP_ROOT/codex/skills" "$TMP_ROOT/claude/skills"
  cp -R reviewing-article-quality "$TMP_ROOT/codex/skills/reviewing-article-quality"
  cp -R reviewing-article-quality "$TMP_ROOT/claude/skills/reviewing-article-quality"
  test -f "$TMP_ROOT/codex/skills/reviewing-article-quality/SKILL.md"
  test -f "$TMP_ROOT/claude/skills/reviewing-article-quality/references/rubric.md"
  rm -rf "$TMP_ROOT"
  ```

  预期：命令退出码为 `0`。

- [x] **Step 4: 检查 Markdown 和 Git diff**

  运行：

  ```bash

  rg -n 'TBD|TODO|待补充|稍后填写' README.md && exit 1 || true
  ```

  预期：没有尾随空格、冲突标记或占位内容，命令退出码为 `0`。
